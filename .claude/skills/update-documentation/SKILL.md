name: update-documentation
description: Creates/updates project documentation based on the changes in the current branch (compared to master). Spawns three codebase explorer agents to summarize each section of the codebase, followed by the `documenter` agent to update all documentation based on the diff content and summaries generated by the explorers. This is meant to be run at the end of a phase and/or sub-phase from the project roadmap.
disable-model-invocation: true
effort: xhigh
---

You are the orchestrator for a documentation update based on the current branch. Assume all development work is complete. Spawn the three agents listed below to summarize each area of the codebase, get the complete list of changed files and code changed in each file, then spawn the `documenter` agent to update the project documentation based on the git diff and explorer summaries.

---

# Step 1 - Confirm branch

```bash
git branch --show-current
```

If on `master`, stop and report: "Checkout a feature branch before running /update-documentation."

---

# Step 2 - Spawn the explorer agents

Delegate documentation extraction sequentially.

Spawn the following agents in order, one at a time:
1. `devops-explorer`
2. `backend-explorer`
3. `frontend-explorer`

Each agent is responsible for generating documentation for its own area. The agents should return structured markdown summaries. Do not proceed until all three have completed.

---

# Step 3 - Spawn the documenter agent

First, get the changed files:
```bash
git diff master...HEAD -- 
```

Then, spawn the `documenter` agent, passing the git diff and all 3 summaries generated from the explorer agents, along with the following instructions:

**Instructions:**
Review the following changes on branch `[current-branch]` compared to `master`. You are in **scoped mode** - review ONLY the diff content and summaries provided below, and use that content to update project documentation and verify consistency.

Changed files:
`[the full list of files changed]`

Diff:
`[diff content]`

Explorer summaries:
`[the structured markdown summaries provided by the explorer agents]`

Remember to stick to your scope limitation and update only documentation relevant to the code changes provided.

---

# Step 4 - Summarize

Return a summary:

```text
Updated

✓ README.md

✓ docs/api/README.md

✓ docs/components/README.md

✓ docs/components/App.md

✓ docs/hooks/README.md

✓ docs/hooks/useGenerateTickets.md

✓ docs/services/README.md

✓ docs/services/api.md

✓ docs/infrastructure/README.md

✓ docs/infrastructure/docker.md

✓ docs/infrastructure/nginx.md

Documentation updated successfully.
```
